# Voice, talking-head and lip-sync research log

This repo stores the reproducible part of the research: notebooks, Kaggle scripts, metadata templates and conclusions.

It intentionally does **not** store private portraits, private voice references or generated identity videos.

## Privacy rules

Never commit:

- real face photos such as `portrait.jpg` or `avatar.jpg`
- real voice references such as `reference_voice.wav`, `reference_qwen_icl.wav` or `audio.wav`
- generated videos with identifiable face/voice
- Kaggle `output-current/`, `error-current/`, `kaggle-runs/` downloads or logs

Commit only:

- scripts
- notebooks
- metadata templates
- placeholder assets
- general research notes

## How to run any working kernel

From a kernel folder:

```bash
cp kernel-metadata.template.json kernel-metadata.json
```

Edit:

```json
"id": "<kaggle-username>/<private-kernel-slug>",
"dataset_sources": ["<kaggle-username>/<private-assets-dataset>"]
```

Then push:

```bash
kaggle kernels push
# or, in this OpenClaw setup:
kaggle-oc kernels push
```

Recommended accelerator: GPU.

After completion, download outputs with Kaggle CLI if needed, but do not commit them.

## Research timeline and conclusions

### 1. Basic voice clone + Wav2Lip

First working direction: generate cloned speech, then use Wav2Lip to animate a static portrait.

Technologies:

- Coqui XTTS v2
- Wav2Lip
- FFmpeg recoding

What worked:

- End-to-end pipeline was feasible on Kaggle.
- Wav2Lip could sync mouth movement from generated audio.

Reflection:

- Useful as a proof of concept.
- Mouth movement is often too synthetic, and full-face motion is weak because Wav2Lip mainly targets the mouth region.
- Good fallback when speed matters, not the strongest visual result.

### 2. Voicebox / Chatterbox candidate search

Next step: compare voice candidates before committing to video generation.

Technologies:

- Voicebox-style candidate generation
- Chatterbox Turbo English
- Chatterbox Multilingual Spanish
- LuxTTS attempt path
- Qwen3-TTS candidate path

Working notebook:

- `notebooks/voicebox_candidates_kaggle.ipynb`

What worked:

- Chatterbox Turbo produced a usable English candidate.
- Candidate generation as a separate step is useful because voice quality should be judged before spending time on video.

Reflection:

- Voice candidate search should remain separate from video generation.
- Best workflow: generate several candidate audios, listen manually, then feed the best one into Ditto, MuseTalk, SadTalker or Wav2Lip.

### 3. Voicebox/Chatterbox audio + Wav2Lip

After choosing the best Chatterbox candidate, it was passed into Wav2Lip.

Technologies:

- Chatterbox Turbo generated audio
- Wav2Lip mouth animation
- CPU-forced Wav2Lip path on Kaggle to avoid GPU compatibility issues
- FFmpeg H.264/yuv420p output

Working notebook:

- `notebooks/voicebox_chatterbox_wav2lip_kaggle.ipynb`

What worked:

- Clean pipeline from generated candidate audio to a final MP4.
- Useful baseline for audio-driven mouth movement.

Reflection:

- Video quality is limited by Wav2Lip. The audio candidate may be good, but the resulting face animation is not the best overall.
- Keep it as a quick comparison baseline.

### 4. Qwen3-TTS 0.6B x-vector mode + Wav2Lip

We tested Qwen's smaller voice clone path with x-vector style cloning.

Technologies:

- `Qwen/Qwen3-TTS-12Hz-0.6B-Base`
- x-vector clone mode
- Wav2Lip

What worked:

- Generated speech and video were produced.

Reflection:

- x-vector-only style cloning was less convincing than ICL mode with exact reference text.
- This led to the next step: cleaner reference audio plus exact transcript.

### 5. Qwen3-TTS 0.6B ICL + Wav2Lip

The better Qwen strategy used a clean reference recording with exact reference text.

Technologies:

- `Qwen/Qwen3-TTS-12Hz-0.6B-Base`
- ICL voice clone mode
- exact `ref_text`
- Wav2Lip

Working notebook:

- `notebooks/qwen_icl_wav2lip_kaggle.ipynb`

Key implementation detail:

```python
model.generate_voice_clone(
    text=TARGET_TEXT,
    language="English",
    ref_audio="reference_qwen_icl.wav",
    ref_text=reference_text,
    x_vector_only_mode=False,
)
```

What worked:

- Better speaker similarity than the earlier x-vector-only direction.
- A clean 30-45 second reference with accurate transcript is a strong practical baseline.

Reflection:

- For Qwen, reference transcript quality matters a lot.
- Use ICL mode when possible.

### 6. Qwen ICL + SadTalker long video

After getting better Qwen audio, we used SadTalker for a longer generated video.

Technologies:

- Qwen3-TTS 0.6B ICL voice cloning
- SadTalker
- Kaggle GPU
- FFmpeg H.264 output

Working notebook:

- `notebooks/qwen_icl_sadtalker_long_kaggle.ipynb`

What worked:

- Produced a long video successfully.
- SadTalker was robust once compatibility patches were applied.

Reflection:

- SadTalker is not the best mouth realism, but it is stable and can handle longer clips.
- Disable GFPGAN enhancer for longer videos if memory becomes tight.

### 7. SadTalker from existing audio

We also kept a simpler SadTalker path for when audio already exists.

Technologies:

- SadTalker
- private `portrait.jpg`
- private `audio.wav`

Working notebook:

- `notebooks/sadtalker_from_audio_kaggle.ipynb`

What worked:

- Straightforward animation from existing audio.

Reflection:

- Useful fallback and debugging baseline.
- Less convincing than Ditto for the best short talking-head result.

### 8. AniPortrait

AniPortrait generated output, but the useful final video had to be cropped out of a debug/comparison layout.

Technologies:

- AniPortrait
- audio-driven portrait animation
- FFmpeg crop from right panel

What worked:

- It produced a final animated face region.

Reflection:

- Not selected as a final notebook because the result was weaker and the default output format was inconvenient.
- Mouth quality was not better than Ditto, so it is documented as explored but not kept as a recommended working baseline.

### 9. Ditto exp065, best visual baseline

Ditto became the best observed visual/lip-sync baseline.

Technologies:

- `antgroup/ditto-talkinghead`
- PyTorch checkpoints from `digital-avatar/ditto-talkinghead`
- expression scaling through `use_d_keys`
- FFmpeg H.264/yuv420p recoding

Working notebook:

- `notebooks/ditto_exp065_kaggle.ipynb`

Key setting:

```python
'use_d_keys': {
    'exp': 0.65,
    'pitch': 1.0,
    'yaw': 1.0,
    'roll': 1.0,
    't': 1.0,
}
```

What worked:

- Best result in this pass.
- Natural head movement.
- Better overall face animation than Wav2Lip and SadTalker.
- Reducing expression movement to `0.65` helped reduce mouth deformation.

Reflection:

- Keep Ditto exp065 as the benchmark to beat.
- Future sweeps should test `exp` around `0.55-0.75`, crop scale and portrait framing.

### 10. MuseTalk v1.5 fullfix

MuseTalk eventually worked after making the environment and asset setup deterministic.

Technologies:

- MuseTalk v1.5
- Python 3.10 venv inside Kaggle
- Torch `2.0.1+cu118`
- MMLab stack: `mmcv==2.0.1`, `mmdet==3.1.0`, `mmpose==1.1.0`
- DWPose checkpoint
- SD VAE
- Whisper tiny

Working notebook:

- `notebooks/musetalk_fullfix_kaggle.ipynb`

Best setup strategy:

- Do not rely only on MuseTalk's download script.
- Force-download and verify all runtime assets before inference.
- Install `mmpose` without the fragile `chumpy` build dependency and provide a small optional-import stub.

What worked:

- Clean final MP4 from private portrait and private audio.
- No debug split-screen.

Reflection:

- MuseTalk is worth keeping as a modern baseline.
- Setup is more fragile than Ditto, but now reproducible in the checked-in notebook/script.
- Future work should tune audio padding, cheek width, crop, portrait framing and parsing mode.

## Current ranking

1. **Ditto exp065**: best visual result so far.
2. **MuseTalk fullfix**: promising modern baseline, now reproducible.
3. **Qwen ICL + SadTalker**: best robust long-form path.
4. **Qwen ICL + Wav2Lip**: useful voice/lip-sync baseline, limited face realism.
5. **Voicebox/Chatterbox + Wav2Lip**: useful for candidate audio testing, video quality limited by Wav2Lip.
6. **AniPortrait**: explored, not recommended as a main baseline from this pass.
